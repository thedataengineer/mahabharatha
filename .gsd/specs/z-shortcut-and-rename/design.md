# Technical Design: z-shortcut-and-rename

## Metadata
- **Feature**: z-shortcut-and-rename
- **Status**: DRAFT
- **Created**: 2026-01-30

---

## 1. Overview

### 1.1 Summary
Two coordinated changes: (1) generate `z:*.md` shortcut files that redirect to `mahabharatha:*.md` slash commands, update install/uninstall to handle both prefixes; (2) rename `troubleshoot` → `debug` across all code, commands, docs, and tests (26+ files).

### 1.2 Goals
- `/z:kurukshetra` works identically to `/mahabharatha:kurukshetra` for all 20 commands
- `mahabharatha debug` replaces `mahabharatha troubleshoot` at CLI
- Zero behavioral regression

### 1.3 Non-Goals
- Changing any diagnostic logic or test assertions
- Adding new troubleshoot/debug features
- Backward compat alias (old name simply stops existing)

---

## 2. Architecture

### 2.1 `/z` Shortcut Approach

Each `z:*.md` file is a thin redirect that includes the full `mahabharatha:*.md` content. Implementation:

```
# mahabharatha/data/commands/z:kurukshetra.md (generated)
<!-- Z-SHORTCUT: This is a shortcut for /mahabharatha:kurukshetra -->
<!-- Auto-generated by MAHABHARATHA. Do not edit directly. Edit mahabharatha:kurukshetra.md instead. -->

<command-name>/z:kurukshetra</command-name>
<import>mahabharatha:kurukshetra</import>
```

However, Claude Code slash commands don't support `<import>`. The practical approach:

**Option A**: Duplicate content (bloat, drift risk)
**Option B**: Generate `z:*.md` at install time by copying content with `mahabharatha:` → `z:` in the command-name tag
**Option C**: Symlink `z:kurukshetra.md` → `mahabharatha:kurukshetra.md` (simplest, no drift)

**Decision**: **Option C — Symlinks**. `install-commands` already uses symlinks. Each `z:*.md` symlinks to the corresponding `mahabharatha:*.md` source file. The slash command system reads the file content regardless of name. The file _name_ determines the command prefix (`z:kurukshetra` vs `mahabharatha:kurukshetra`), but the _content_ is shared.

### 2.2 Install Commands Changes

```python
# Current: COMMAND_GLOB = "mahabharatha:*.md"
# New: Also generate z:*.md symlinks pointing to mahabharatha:*.md source files

def _install(...):
    # 1. Install mahabharatha:*.md as before
    # 2. For each mahabharatha:*.md, create z:*.md symlink → same source
```

### 2.3 Rename Strategy

Pure file rename + find-and-replace. No architectural changes.

**Rename map:**
| From | To |
|------|----|
| `mahabharatha/commands/troubleshoot.py` | `mahabharatha/commands/debug.py` |
| `mahabharatha/data/commands/mahabharatha:troubleshoot.md` | `mahabharatha/data/commands/mahabharatha:debug.md` |
| `tests/unit/test_troubleshoot_cmd.py` | `tests/unit/test_debug_cmd.py` |
| `tests/integration/test_troubleshoot.py` | `tests/integration/test_debug.py` |

## 3. Key Decisions

### Decision: Symlinks for `/z` shortcuts
**Context**: Need `/z:kurukshetra` to behave identically to `/mahabharatha:kurukshetra`.
**Options**: (1) Duplicate files, (2) Generate at install, (3) Symlink
**Decision**: Symlink. `z:kurukshetra.md` → `mahabharatha:kurukshetra.md` in the install target directory.
**Rationale**: Zero drift, zero maintenance, install-commands already uses symlinks.

### Decision: No backward compat for `troubleshoot`
**Context**: Keep old name as alias?
**Decision**: No. Clean break. `mahabharatha troubleshoot` → error.
**Rationale**: Backlog item explicitly says "rename", not "alias". Keeping both adds confusion.
